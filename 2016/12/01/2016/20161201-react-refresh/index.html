create-react-app

create-react-app is a turbo app. That means there are 20 plus features that are supported out of the box without additional config. Just to highlight a few:

  • webpack: dev server, hot reload, production build
  • npm scripts: start, test, build
  • update to a new release
  • import images and fonts
  • proxy api requests
  • using https in dev
  • tests with a choice of lib
  • code coverage
  • continuous integration
  • deploy: github, surge, and more

Understand the React Component Lifecycle Methods

  • componentWillMount: once before render
  • render: draws to DOM
  • componentDidMount: once after render

Manage React Component State with Lifecycle Methods

  • componentWillMount: access to state/props but not DOM
  • componentDidMount: access to DOM (use ReactDOM.findDOMNode())
  • componentWillUnmount: do cleanup (i.e. clearInterval(this.myInterval))

Control React Component Updates When New Props Are Received

  • onClick re-render with new props
  • componentWillReceiveProps sets increasing for all but first click
  • shouldComponentUpdate constrains rendering to once every 5 clicks

Understand JSX at a Deeper Level

  • components and tags compile to different results
  • self close both normal and custom components
  • remember to return one node
  • props won’t render unless data-myprop or my-prop (dash in name)
  • comments in jsx suffix of the transpiled code {/* this is a comment */}
  • use ternary: {i>1 ? ‘many’ : ‘one’}
  • use short-circuit: {i>1 && ‘many’ }

Understand React.Children Utilities

iterate over children of components

  • React.Children.map(this.props.children, (items, i) => {}) // handles one item

  • React.Children.toArray(this.props.children) // return array even if one child

  • React.Children.forEach(this.props.children, fn)

  • React.Children.only(this.props.children, fn) // throws when more than one

Use React.cloneElement to Extend Functionality of Children Components

  • common pattern for things like tab interface or grouped buttons
  • when mapping over children it does not provide the child component only a descriptor. Use React.cloneElement(child, propertyObj) to get to the components
  • here we apply an onClick property to each button
