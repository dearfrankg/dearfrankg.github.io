Ranking Dev Tools

Web Dev Tools by Preference

Web App Creation Frameworks

#1 nwb

nwb (node webpack and babel)

  • produces apps, components and libraries.
  • supports react/preact/inferno/react-native
  • quick-start with a single .js file.
  • optional configuration
  • automatic dependency installation
  • thought out skeleton supports delivery to npm
  • includes testing and coverage support

#2 create-react-app

  • produces react apps
  • zero config
  • extensible with custom scripts
  • includes top notch testing and coverage support
  • supported by facebook

#3 fuse-box-create-react-app

  • a create-react-app clone with fast babel compilation.
  • uses fuse-box fast babel compiler under the hood.

State Management By Preference

#1 Mobx

  • best of breed
  • easy to get started
  • medium learning curve
  • good support and fan following

Mobx has taken over the number 1 spot.

I agree with the philosophy of Mobx that there is one source of truth that everything else can be derived from. That includes being able to derive actions and reducers.

Mobx is easier to implement and provide 100% of the functionality of Redux. Make sure you look into the opinionated way to do Mobx. Start with the mobx-state-tree.

#2 Redux

  • most popular at this time
  • medium learning curve
  • mega community support and libs

Desktop Dev Tools by Preference

#1 Electron

The electron framework has been gaining steam for a few years and has been used to create the following apps:

  • atom
  • slack
  • MS Dev Studio
  • webtorrent

I don’t support any electron app creation frameworks at this time.
That means create with just node, babel and electron.
